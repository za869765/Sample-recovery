# 腸胃篩檢體回收追蹤系統 — 設計需求

## 1. 系統定位

**主體**：桌機網頁 UI，用於行動醫院後續的腸/胃篩檢體回收追蹤管理。
**後援**：Google Sheets 作為持久化儲存（DB），不是主要操作介面。

操作便利性最重要。試算表只是後援，所有日常操作（編輯個案狀態）都在 UI 完成。

**目標使用者**：1 位護理師（單人使用）。
**裝置**：純桌機。不需要響應式設計、不考慮手機/平板。

---

## 2. 來源檔案（Input）

每場行動醫院結束後會有 4 個 Excel 檔（檔名格式不固定，需依**內容欄位 header** 自動識別）：

### 2.1 fobt45-75歲名冊 (.xls)
**識別特徵**：欄位含 `CISID`, `姓名`, `身份證字號`, `聯絡電話`, `手機`（共 25 欄）

範例欄位：
```
CISID, 支付方式, 醫令代碼, 姓名, 性別, 出生日期, 身份證字號, 聯絡電話, 手機, 地址,
教育程度, 父母兄弟姊妹子女有無大腸癌, 父母兄弟姊妹子女有無其他癌症, 請問您有無腸道不適症狀,
檢查醫療院所代碼, 門診日期, 糞便潛血檢驗機構代碼, 檢驗日期, 檢查結果, 檢驗結果數值,
試劑商品名, 其他定量試劑商品名稱, 前項檢查陽性個案轉至醫院代碼, 前項檢查陽性個案轉診醫院電話, age
```

**內容意義**：腸篩（FOBT）已掛已發試劑的人。雖然檔名是 45-75，實際範圍是 **40~75 歲**（含 40-44 歲簽家族史切結書者）。可能包含「不在衛生所查詢」的專程來做者。健保申報。

### 2.2 胃癌名冊 (.xls)
**識別特徵**：欄位 `IDNO, cisid, NAME, jdate, age, BIR_Y, BIR_M, BIR_D, SEX, HomeAddr, HomeTEL1, HomeTEL2, MOBTEL1`（13 欄、全英文）

**內容意義**：胃癌篩檢已掛已發試劑的人，純 **45~75 歲**。健保申報。

### 2.3 非45-75歲FOBT名冊 (.xls)
**識別特徵**：欄位只有 `name, id93, sdate93, sex, age`（5 欄）

**內容意義**：年齡**不在 45-75** 範圍但行動醫院當天有掛號者的候選清單。其中可申請市府補助（不申報健保）做 FOBT 的範圍是 **30~39 歲 OR ≥76 歲**。這個檔本身**缺身分證/電話**，需用 `id93` 配對「衛生所查詢」補齊。

### 2.4 衛生所查詢 (.xls)
**識別特徵**：欄位 `idno, name, id93, expr1003, tel1, tel2, cellular, bir_y, bir_m, bir_d, sex, address, AGE, sdate93, adult, adultdate, pap, papdate`（18 欄）

**內容意義**：當天行動醫院全部掛號名單。`id93` 是行動醫院編號（13 碼，例 `2026050501001`），前 8 碼是西元 yyyymmdd（但日期不可信，請用 `sdate93` 民國 YYYMMDD 為準）。

## 3. 業務邏輯：三大族群 → 寫入試算表

| # | 族群 | 來源 | 申報 | 寫入分頁 |
|---|---|---|---|---|
| 1 | 腸篩 **45~75 歲** | fobt45-75歲名冊 | 健保 | 行醫腸篩{日期} |
| 2 | 腸篩 **40~44 歲家族史**（切結書） | fobt45-75歲名冊（HIS 已包含） | 健保 | 行醫腸篩{日期} |
| 3 | 腸篩 **30~39 OR ≥76 + 市府補助** | 非45-75歲FOBT 中**人工勾選**的人 | 不報健保 | 行醫腸篩{日期} |
| 4 | 胃篩 **45~75 歲** | 胃癌名冊 | 健保 | 行醫胃篩{日期} |

### 3.1 自動寫入
- 腸篩分頁 = fobt45-75歲名冊全部 + 勾選的非45-75歲FOBT
- 胃篩分頁 = 胃癌名冊全部

### 3.2 非45-75歲FOBT 的處理
- 候選清單 = 整個非45-75歲FOBT 名冊（74 人左右）
- UI 上列出，護理師**點按編號**選擇誰實際做了（市府補助）
- 沒勾選的不寫入試算表（視同拒做）
- 40~44 歲非家族史者本來就無法做（不在補助範圍也不在 fobt45-75）→ UI 上可以標灰但仍列出，避免漏掉

### 3.3 補齊身分證/電話
- 非45-75歲FOBT 缺 `idno / tel1 / cellular` → 用 `id93` 對 衛生所查詢
- 規則：一定能對到（業務保證沒有例外）。對不到要警告。

## 5. UI 流程

### 5.1 進入頁（首頁）
兩個入口（大按鈕）：
1. **讀取行醫專案** — 列出雲端已存在的場次（按日期降冪），點一場進入該場個案管理頁
2. **開啟新行醫專案** — 進入「上傳初始化」流程

### 5.2 開啟新行醫專案（初始化流程）

**步驟 1：單一檔案上傳區**
- 一個大區域，可一次拖入或選 4 個檔（也支援逐一加入）
- 系統依**欄位 header 內容**自動識別這 4 個檔的類型（不靠檔名）
- 識別結果即時顯示在區域內：`✓ fobt45-75歲名冊 (104 筆)` `✓ 胃癌名冊 (81 筆)`...
- 未識別出來的檔以紅字標示「無法辨識」
- 4 個都到齊才能進下一步

**步驟 2：自動比對結果摘要**（保留現有設計，數字會跟新邏輯）
- 腸篩 fobt45-75（含 40-44 家族史）人數
- 胃篩 胃癌名冊 人數
- 衛生所查詢 總筆數
- 非45-75歲FOBT 候選人數
- 偵測到的行動醫院日期 + 將建立的分頁名

**步驟 3：勾選免掛號做（非45-75 候選人）**
- 表格列出非45-75歲FOBT 全部人
- 每行顯示：編號（13 碼或末 5 碼，可點按的圖示）、姓名、身分證（從衛生所查詢補的，缺則顯示「-」並標紅）、性別、年齡
- **點編號圖示** = 切換「勾選為市府補助做了」狀態（再點取消）
- 30-39 OR ≥76 歲是可申請補助範圍，視覺上突出（如綠色邊框）
- 40-44 歲（無家族史）不能做，標灰但仍可看到

**步驟 4：確認 + 初始化**
- 顯示最終要寫入的人數：腸篩 X 人 / 胃篩 Y 人
- 按「建立場次」→ 寫入試算表（建分頁、寫入個案基本資料+發管日，狀態欄留空）
- 完成後自動跳到該場次的「個案管理頁」

### 5.3 個案管理頁（核心 UI）

**頁面上方控制列**：
- 場次名稱（如「1150516 場」）+ 切換場次下拉
- **回收日期選擇器**（MM/DD 格式）— 預設帶今天，可改；值需持久化（下次打開記得）；**不寫入試算表**，只用於批次操作
- 分頁切換：腸篩 / 胃篩
- 篩選器：未回收 / 已回收 / 全部 / 待轉介 / 取消追蹤 等
- 搜尋框（姓名/身分證/編號）

**主表格**（UI 欄位順序，與試算表欄位順序不同）：

| UI 欄 | 標題 | 互動 |
|---|---|---|
| 1 | 編號（13 碼，例 `2026050501013`，顯眼可點按）| **點 = 設「檢體已回收」+ 送管日=上方選的回收日**；視覺切換已回收/未回收 |
| 2 | 姓名 | 唯讀 |
| 3 | 身分證 | 唯讀 |
| 4 | 手機1 | 可改 |
| 5 | 手機2 | 可改 |
| 6 | 檢體狀態 | 下拉/勾選（已回收/未回收/...）|
| 7 | 報告狀態 | 下拉（正常/異常/...）|
| 8 | 待轉介 | 勾選 |
| 9 | 轉介未做腸鏡 | 勾選 |
| 10 | 發管日 | 預壓行動醫院日期，可改 |
| 11 | 送管日 | 預設空白，回收時帶入 UI 回收日，可改 |
| 12 | 取消追蹤 | 勾選 |

**核心互動：批次回收**
1. 護理師在上方選好今天「回收日期」(05/22)
2. 桌上一疊試管送回來，每個試管上的編號（13 碼）對應 UI 一行
3. 在 UI 表格上**連續點各行的編號**：每點一個 → 該行檢體狀態變「已回收」+ 送管日填 05/22 + 視覺變灰
4. 同步寫入試算表（每點一次就寫一次，不需按「儲存」）

**便利性要求**：
- 連續點按要快（一次 click 完成所有狀態更新）
- 視覺變化明顯（已回收的列換背景色）
- 防誤點：點到已回收的編號 → 確認後取消回收（變回未回收）
- 鍵盤支援：搜尋框聚焦時可直接輸入編號末 5 碼 → 自動定位該行 → Enter = 標記回收（更快）
- 表頭固定（scroll 時不動）
- 計數顯示：未回收 / 已回收 / 待轉介 / 等的即時統計

## 6. 額外要點

### 6.1 同步策略
- 試算表只是後援，UI 操作要先在前端立即反應（樂觀更新）
- 背景非同步同步到試算表
- 同步失敗要有重試 + 視覺提示（網路斷線時可離線編輯）

### 6.2 場次列表
- 「讀取行醫專案」頁 = 從試算表讀所有分頁名（行醫腸篩/胃篩 開頭的）→ 按日期分組
- 每場顯示：日期、腸/胃 各 N 筆、回收進度（已回收/全部）

### 6.3 不需要的功能
- 不需手機/平板適配
- 不需多人協作（單人使用）
- 不需歷史變更追蹤（修改直接覆蓋試算表）
